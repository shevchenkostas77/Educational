Коллекции - это какой-то контейнер, совокупность объектов, которые хранятся вместе.
Некоторые коллекции могут содержать дубликаты, а некоторые нет.
Есть коллекция List, Queue, Set которые происходят от интерфейса Collection <- Iterable
Есть еще одна коллекция, которая не происходит от интерфейса Collection - Map.
Интерфейс Мар всегда рассматривают как коллекцию.
Все коллекции работают с generics, т.е. они содержат элементы определенного вида.
По иерархии понятно, что List, Queue, Set происходят от Iterable, благодаря интерфейсу Iterable
есть возможность «пробегаться» по элементам коллекции с помощью for each loop.
Методы, которые есть в интерфейсе Collection будут общими для всех классов,
которые его имплементируют (add(), remove(), size(), isEmpty()).
Есть интерфейс Collection, а есть класс Collections – это разное.
Collections – это utility class (вспомогательный класс), который содержит
статические методы для работы с коллекциями.
List – это базовый интерфейс для коллекции, который содержит упорядоченные элементы.
Каждый элемент имеет индекс. Может содержать дубликаты и null.

Классы интерфейса List:
-ArrayList;
-LinkedList;
-Vector и его subclass Stack;
